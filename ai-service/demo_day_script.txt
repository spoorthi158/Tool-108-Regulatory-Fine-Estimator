QUESTION 2:
"What happens if the AI service goes down?"

ANSWER:
"We have a three-layer protection system.

Layer 1 — We retry the Groq API up to 3 times
with exponential backoff before giving up.

Layer 2 — We have an 8-second timeout on every
Groq call so the service never hangs.

Layer 3 — If all retries fail, we return a
pre-built professional fallback response
with is_fallback set to true.

The Java backend always receives a valid response.
The tool never shows an error to the user.
The tool never returns HTTP 500."

------------------------------------------------------------

QUESTION 3:
"How do you prevent prompt injection attacks?"

ANSWER:
"We built a custom sanitiser in services/sanitiser.py
that runs on every input before it reaches the AI.

It does two things:
First — it strips all HTML tags from the input
to prevent cross-site scripting.

Second — it checks for 10 known injection patterns
such as 'ignore previous instructions',
'act as', 'jailbreak', and 'system prompt'.

If any injection pattern is detected, we return
HTTP 400 with an error message immediately —
the input never reaches Groq.

We verified this with OWASP ZAP scanning —
zero Critical and zero High security findings."
